About

This app demonstrates a simple CRUD flow for a tea collection using a backend API and an HTML form. The form stores an ID for edits so that the frontend can update or delete specific tea entries through the API.

Learning Prompts

Can you add a connection string to the NodeJS environment (e.g. .env file) for your own MongoDB Atlas Cluster?

Use MongoDB Compass to verify that your data collection form is working.

Try out all CRUD operations: Create a new tea, Edit an existing tea by clicking the Edit button, and Delete a tea you no longer need.

Explore the API endpoints in your browser: visit /data to see all teas in JSON format, or try /search?terms=Sencha to search for specific teas.
